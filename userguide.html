먼저 mock으로 계약을 확인하고, Hermes나 Codex는 의도할 때만 쓴다.

이 가이드는 현재 코드에서 실제 지원하는 명령만 다룬다. 모델 호출 경로는 로컬 CLI 인증 상태에 의존하며, 결과는 투자 조언이 아니라 리서치 보조 자료다.

installeditable package로 CLI를 등록한다.
doctorrunner executable과 smoke 경로를 확인한다.
analyzeshallow pipeline을 실행한다.
resumemanifest graph shape를 유지하며 이어간다.

1. 설치

repository root에서 실행한다.

Editable install

python -m pip install -e .

Convenience runner

./run_stock_agents.sh NVDA
./run_stock_agents.sh ask

`run_stock_agents.sh`는 `.venv`와 yfinance 의존성을 확인하고, 티커 shortcut은 기본적으로 Codex runner를 사용한다.

2. 환경 확인

`doctor`는 executable 확인과 runner별 smoke를 담당한다.

빠른 확인

no model call
stock-agents doctor --smoke-runner none
stock-agents doctor --smoke-runner mock

Mock smoke는 로컬 fixture output을 검증한다. Hermes smoke는 command shape를 확인하지만 live LLM을 호출하지 않는다.

실제 runner 확인

auth dependent
stock-agents doctor --smoke-runner hermes
stock-agents doctor --smoke-runner codex

Codex는 `codex exec`가 실제로 성공해야 usable로 본다. `codex login status`만으로는 충분하지 않다.

3. 분석 실행

기본 회귀 확인은 mock runner를 사용한다.

Deterministic mock run

stock-agents analyze SPY   --date 2026-01-15   --runner mock   --language Korean   --analysts market,news   --debate-rounds 1   --risk-rounds 1   --depth shallow

명령은 run directory, Markdown final report path, HTML final report path를 출력한다.

All analysts plus multi-round debates

bounded 1..3
stock-agents analyze SPY   --date 2026-01-15   --runner mock   --analysts all   --debate-rounds 2   --risk-rounds 2

`all`은 market, sentiment, news, fundamentals를 선택한다. Round count가 1보다 크면 반복 role task id에 `_roundN` suffix가 붙는다.

Hermes run

live model
stock-agents analyze SPY   --date 2026-01-15   --runner hermes   --provider openai-codex   --model gpt-5.5   --timeout-seconds 240   --language Korean   --analysts market,news   --depth shallow

Hermes auth는 로컬 Hermes Agent 설치가 담당한다. Routine regression에는 mock을 우선 사용한다.

Codex run

codex exec
stock-agents analyze SPY   --date 2026-01-15   --runner codex   --model gpt-5.5   --reasoning-effort medium   --timeout-seconds 240   --language Korean   --depth shallow

Codex logs are stored under `logs/codex/` inside the run directory.

4. 재개

Resume은 manifest에 기록된 graph-shaping 옵션을 바꾸지 않는다.

Resume a run

stock-agents resume runs/SPY/2026-01-15/<run_id>   --runner mock   --language Korean
  • Omit `--analysts`, `--debate-rounds`, and `--risk-rounds` to reuse manifest values.
  • If you pass one of those options and it differs from `manifest.json`, resume fails before new tasks run.
  • Start a new run when you want to change graph shape.

5. 검증과 검사

생성된 파일은 CLI로 다시 읽고 검증할 수 있다.

Show run

stock-agents show-run runs/SPY/2026-01-15/<run_id>
checkpoint status, current step, output aliases를 출력한다.

Validate output

stock-agents validate ... --role portfolio_manager
raw stdout에서 JSON을 추출하고 Pydantic schema로 검증한다.

Build task

stock-agents build-tasks SPY --date 2026-01-15 --role market_analyst
단일 task package를 stdout으로 렌더링한다.

7. 주의사항

현재 지원 범위를 넘는 사용은 문서화된 기능이 아니다.

Only shallow

`--depth shallow`만 구현되어 있다. standard/deep 분석은 아직 없다.

Market data

Market facts는 yfinance로 최소 1년 범위의 일봉을 수집한다. 실패하거나 비활성화된 경우 252개 weekday fixture bar로 대체된다.

Research only

최종 report는 투자 조언이 아니다. 모든 데이터와 판단은 독립적으로 검증해야 한다.

Visual thesis: dark analyst ops, dense command-first guide, teal status signal. News uses yfinance first, with Naver fallback for sparse Korean ticker coverage.
